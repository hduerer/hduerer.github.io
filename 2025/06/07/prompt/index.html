Prompt

Prompt 是什么？

提示词（Prompt）是用户输入给大语言模型（LLM, Large Language Model）的文本指令，用于引导模型生成期望的输出，决定了模型响应的上下文、任务类型、语言风格、格式要求等。简单来说，就是你向模型提供的 “提示词”。

提示词是由一些要素组成的，提示词可以包含以下任意要素：

  • 指令：想要模型执行的特定任务或指令。

  • 上下文：包含外部信息或额外的上下文信息，引导语言模型更好地响应。

  • 输入数据：用户输入的内容或问题。

  • 输出指示：指定输出的类型或格式。

Prompt 种类?

提示词类型 说明 示例
System Prompt 系统级指令，设定行为规则 “你是专业的翻译器，只输出中译英结果”
User Prompt 用户输入内容，用于触发模型响应 “请解释一下量子力学的基本概念”
Assistant Prompt 模型历史响应记录（可作为上下文） “量子力学研究微观粒子行为…”
Few-shot Prompt 举例型提示词，引导模型模仿特定格式或样式 “Q: … A: …”
Instruction Prompt 指令型提示词，如“请扮演…” “以表格形式列出…” “请你用表格总结中国朝代”
Role Prompt 给模型设定身份或角色 “你现在是一位心理咨询师”

Prompt Engineering

定义：Prompt Engineering 是设计和优化输入提示（prompt）以获得预期输出的过程。在与大型语言模型交互时，如何构造提示会显著影响模型的回答质量。

项目地址(Stars56.9k): https://github.com/dair-ai/Prompt-Engineering-Guide?tab=readme-ov-file

Prompt Engineering Guide: https://www.promptingguide.ai/zh

LLM参数设置

使用提示词时，我们通常会通过 API 或直接与大语言模型进行交互。我们可以通过配置一些参数以获得不同的提示结果。以下是使用不同LLM提供程序时会遇到的常见设置：

1. Temperature

说明：简单来说，temperature的参数值越小，模型就会返回越确定的一个结果。如果调高该参数值，大语言模型可能会返回更随机的结果，也就是说这可能会带来更多样化或更具创造性的产出。

取值范围：0 ~ 2

含义：

  • 低温度（如 0.0）→ 输出更确定、保守
  • 高温度（如 1.5）→ 输出更随机、多样。

以下是DeepSeek官网API官方推荐的参数值: DeepSeek API文档

2. Top_p

说明：使用 top_p（与 temperature一起称为核采样（nucleus sampling）的技术），可以用来控制模型返回结果的确定性。如果需要准确和事实的答案，就把参数值调低。如果寻找更多样化的响应，可以将其值调高点。

取值范围：0 ~ 1

含义：

  • top_p = 1.0：不限制输出多样性。
  • top_p = 0.9：从前 90% 的可能词中随机采样。

注意: 一般建议是改变 Temperature 和 Top_P 其中一个参数就行，不用两个都调整。

3. Max_tokens

token 是模型用来表示自然语言文本的基本单位，可以直观的理解为“字”或“词”；通常 1 个中文词语、1 个英文单词、1 个数字或 1 个符号计为 1 个 token。一般情况下模型中 token 和字数的换算比例大致如下：1 个英文字符 ≈ 0.3 个 token。1 个中文字符 ≈ 0.6 个 token。

说明：限制模型最多生成多少个 token。

注意：总 token 数 = 输入 token + 输出 token，不能超过模型上限。

示例：max_tokens: 100表示最多生成 100 个 token 的内容。

4. Frequency Penalty

说明：frequency penalty 是对下一个生成的 token 进行惩罚， frequency penalty 越高，某个词再次出现的可能性就越小，这个设置通过给重复数量多的 Token 设置更高的惩罚来减少响应中单词的重复。（频率惩罚）

取值范围：0 ~ 2

5. Presence Penalty

说明：presence penalty也是对重复的 token 施加惩罚，但与 frequency penalty不同的是，惩罚对于所有重复 token 都是相同的。出现两次的 token 和出现 10 次的 token 会受到相同的惩罚。 此设置可防止模型在响应中过于频繁地生成重复的词。(出现惩罚)

取值范围：0 ~ 2

说明：鼓励生成新的话题，而不是重复已有的内容。

注意: 与 temperature 和 top_p 一样，一般建议是改变 frequency penalty 和 presence penalty 其中一个参数就行，不要同时调整两个。

6. Stop Sequences

说明：stop sequence 是一个字符串，可以阻止模型生成 token。指定一个或多个“停止词”或字符，遇到这些就停止生成。

类型：字符串或字符串数组

Prompt Engineering技巧

1. 目标明确

技巧：明确你希望模型完成的任务或回答的问题。

示例：

  • 目标不明确：

    "告诉我关于气候变化的事情。"

  • 目标明确：

    "请简要描述气候变化的主要原因及其对农业的影响。"

2. 提供上下文

技巧：为模型提供必要的背景信息或上下文，以帮助其理解任务。

示例：

  • 无上下文：

    "解释一下微积分。"

  • 有上下文

    "作为一名高中生，我正在学习微积分。请用简单的语言解释一下微积分的基本概念。"

3. 做什么还是不做什么?

技巧：避免说不要做什么，而应该说要做什么。

示例：

  • 不做什么：

    "以下是向客户推荐电影的代理程序。不要询问兴趣。不要询问个人信息。客户：请根据我的兴趣推荐电影。代理："

  • 做什么：

    "以下是向客户推荐电影的代理程序。代理负责从全球热门电影中推荐电影。它应该避免询问用户的偏好并避免询问个人信息。如果代理没有电影推荐，它应该回答“抱歉，今天找不到电影推荐。”。``顾客：请根据我的兴趣推荐一部电影。``客服："

4. 提供示例/输出格式

技巧：通过提供示例来展示你期望的输出格式或内容。(和少样本提示相似)

示例：

  • 无示例：

    "生成一个关于产品的报告。"

  • 有示例：

    "生成一个关于产品的报告，格式如下：\n\n- 产品名称：\n- 价格：\n- 特点：\n- 优点：\n- 缺点："

5. 使用分步指示

技巧：对于复杂任务，分解为多个步骤，逐步引导模型完成。

示例：

  • 一步完成：

    "解释并解决这个数学问题：2x + 3 = 7。"

  • 分步指示：

    "首先，解释如何解方程。然后，解方程2x + 3 = 7。"

6. 控制输出长度同时要明确

技巧：通过提示控制输出的长度，确保内容简洁或详细，同时要注意语言要明确。

示例：

  • 无长度：

​ "解释一下量子力学。"

  • 有长度不明确：

    "解释量子力学的基本概念。保持解释简短，只要几句话，不要过于描述。"

  • 有长度明确：

    "用不超过100字解释量子力学的基本概念。"

7. 使用多轮对话

技巧：在需要时，通过多轮对话逐步引导模型生成所需内容。

示例：

  • 单轮对话：

    "告诉我关于Python编程的所有信息。"

4. Tree of Thought (ToT)

定义: ToT 是 “Tree of Thought” 的缩写，它的核心是：把问题的解题过程表示为一个思维树（思路树），通过系统搜索来选择最优路径。不再是一步步贪心地往下走，而是生成多个思路分支、尝试多种想法，然后评估它们再决定如何继续。

原理流程:

  1. 将任务拆解为多个思维步骤（Thought Steps）
  2. 每一步可以扩展为多个候选思路（分支）
  3. 用启发式方法对每条路径评分
  4. 选出最有前途的路径继续展开
  5. 最终找到最优解或多个解的比较

5. ReAct

定义：ReAct 是“Reasoning + Acting”的缩写，核心理念是：将语言模型的思考过程（Reasoning）与对环境的行动（Acting）交替结合，从而更有效地完成任务。类似人类在解决问题时：我们一边思考，一边行动，然后根据行动结果继续思考。

原理流程: 输入 → 思考（Thought） → 执行动作（Action） → 观察结果（Observation） → 再次思考 …

结构化输出

结构化输出在 Prompt Engineering 中是指通过设计特定的提示，引导语言模型生成具有明确格式或结构的输出。这在需要处理数据表格、生成代码、创建报告等任务中尤为重要。通过结构化输出，可以确保生成的内容符合预期的格式，便于后续处理和使用。

结构化输出的常见形式

  1. JSON 格式：适用于需要生成或处理数据对象的任务。
  2. Markdown 格式：用于生成文档或报告，便于阅读和展示。
  3. 表格格式：适用于数据展示和分析。
  4. 代码格式：用于生成特定编程语言的代码片段。

设计结构化输出提示的技巧

  1. 明确格式要求：在提示中清晰地说明输出的格式。
  2. 提供示例：通过示例展示期望的输出格式。
  3. 使用占位符：在提示中使用占位符来指示需要填充的内容。
